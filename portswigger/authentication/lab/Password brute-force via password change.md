# Lab: Password brute-force via password change

## 概要
このLabではパスワード変更の際にブルートフォース攻撃に対する脆弱性を扱う。  
目的は脆弱性を理解し，ブルートフォース攻撃を成功させることである。

---

## 脆弱性の種類
Authentication

---

## 攻撃の結論
パスワード変更画面であえて変更後と確認のパスワードを違うものとし，IF文の論理のミスをつき，ブルートフォース攻撃対策を回避し，
有効なパスワードを明らかにした．

---

### 1. 観察

- ログイン画面に飛ばされたあと合っているパスワードを入力してもはじかれる．（アカウントロックかIPアドレスレート制限の可能性）

### 2. 仮説
- （どんな脆弱性だと考えたか）
- パスワード変更の際にブルートフォース攻撃に対する対策がされていないと予想

### 3. 検証
-ログイン画面でパスワードリセットをわざと間違ったパスワードを入力した．
- Burpで実施した内容：
  - `Repeater` であえて変更後のパスワード二つを変えて送った．
  - Intruderでパスワードの部分をブルートフォース攻撃した，その際あえて変更後のパスワードを違うものにした．
  - （例：有効なユーザー名を特定）
  - （例：その後パスワード候補を `Intruder` で試行）
- 結果：
  - ログイン画面に飛ばされてロックがかけられた（かなり防御が固い）
  - 現在のパスワードが違うとでました．これはロジックミス（まずさきに変更後のパスワードの両方が一致しているか
  - 確認して，そのあと現在のパスワードチェックを行い，その結果を出してそこで処理を終えて失敗のカウントをしなかった．（ロックがかからない答え合わせができる）
  -ひとつだけ変更後のパスワードが違うと出た．（IF文のロジックミス）
