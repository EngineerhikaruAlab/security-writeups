# Lab 001 — Access Control

## Summary
- Vulnerability: Broken access control
- Category: IDOR
- Tool: Burp Suite (Repeater)

## What I observed
The application identifies users by a request parameter.

## Reproduction
1. Open My account
2. Capture request in Burp
3. Send to Repeater
4. Change the identifier
5. Send request

## Why it works
The server trusts user-supplied identifiers and does not verify resource ownership.

## Fix
- Enforce server-side authorization
- Validate resource ownership

## Takeaways
Never trust client-controlled identifiers.
