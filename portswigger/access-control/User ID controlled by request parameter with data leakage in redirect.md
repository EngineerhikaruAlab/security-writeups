# Lab: User ID controlled by request parameter with data leakage in redirect

## 概要
このLabではバックエンドでURLのクエリを偽造したものをはじいてリダイレクトを促す際に生じる脆弱性（EAR：Execution After Redirect）を扱う。  
目的はこの脆弱性を理解し、実際の開発で起こりうるのをイメージすることである。

---

## 脆弱性の種類
Access Control

---

## 攻撃の結論（最短）
有効なアカウントのAPIのGETリクエストのクエリを対象のアカウントにして送る。するとログインページにリダイレクトされるが
その際にバックエンドの処理でプログラムを終了していていないため、対象のアカウントのAPIキーがレスポンスでわかった。
