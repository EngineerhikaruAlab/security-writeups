# Lab: User ID controlled by request parameter

## 概要
このLabではユーザーが変更可能なパラメータがリソースや機能のアクセス制御に使われるときに生じる脆弱性を扱う。  
目的はIDORの攻撃手法を学び、実際の開発でなぜ起きるかなどを理解することである。

## メモ（任意）
- 現代のWeb開発ではAuthenticationフレームワークが全部自動でやってくれるが
- Authorizationはビジネスルールが複雑であるため自動化できない、開発者がすべての機能（URL)に対して手作業で
- If文（所有権チェック）を書かなければならないため、ミスが起きる。
- URL（Uniform Resource Locater)は文字通りどこにあるどのデータ化を指し、HTTPメソッド（やりたい操作）で全部ひっくるめたのがHTTPメソッド
- URLの文字の中に『やりたい操作（deleteやupdate）』を直接書いてしまおう」という設計（アンチパターン）はだめ
